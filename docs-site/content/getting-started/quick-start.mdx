# Quick Start Tutorial

Welcome to the AI Toolkit! This tutorial will guide you through your first document processing workflow in just 10 minutes. By the end, you'll have uploaded a document, processed it with AI, and retrieved the results.

<Callout type="info" title="Before You Start">
Make sure you have completed the [installation guide](/getting-started/installation) and have the AI Toolkit running on your system.
</Callout>

## What You'll Learn

In this tutorial, you'll learn how to:

- Upload your first document via the web dashboard
- Process documents using the REST API
- Analyze content with AI models
- Retrieve and interpret results
- Set up basic workflows

## Step 3: Process Documents via API

Now let's learn how to upload and process documents programmatically using the REST API:

<TutorialStep step={2} title="Using the REST API">

### Upload Document via API

<CodeBlock language="bash" title="Upload Document">
# Upload a document for processing
curl -X POST http://localhost:3000/api/documents/upload \
  -F "file=@sample-portfolio.md" \
  -F "analysisType=full" \
  -F "tags=portfolio,developer,tutorial" \
  -H "Content-Type: multipart/form-data"
</CodeBlock>

Expected response:

<CodeBlock language="json" title="Upload Response">
{
  "success": true,
  "document": {
    "id": "doc_1234567890_abc123",
    "filename": "sample-portfolio.md",
    "type": "md",
    "size": 1024,
    "status": "processing",
    "analysisType": "full",
    "tags": ["portfolio", "developer", "tutorial"],
    "uploadedAt": "2024-01-15T10:30:00Z"
  },
  "message": "Document uploaded successfully and processing started"
}
</CodeBlock>

### Check Processing Status

<CodeBlock language="bash" title="Check Status">
# Replace {document-id} with the actual ID from the upload response
curl http://localhost:3000/api/documents/{document-id}
</CodeBlock>

<CodeBlock language="json" title="Status Response">
{
  "id": "doc_1234567890_abc123",
  "filename": "sample-portfolio.md",
  "status": "completed",
  "analysis": {
    "extractedText": "John Doe - Software Developer...",
    "wordCount": 156,
    "keyTerms": ["software developer", "React", "Node.js", "AWS"],
    "summary": "Portfolio of John Doe, a software developer with 5 years experience...",
    "sentiment": "positive",
    "topics": ["web development", "cloud technologies", "project management"]
  },
  "processedAt": "2024-01-15T10:31:30Z"
}
</CodeBlock>

### List All Documents

<CodeBlock language="bash" title="List Documents">
# Get all documents with pagination
curl "http://localhost:3000/api/documents?page=1&limit=10"
</CodeBlock>

</TutorialStep>

## Step 6: Create Your First Workflow

Workflows allow you to automate document processing tasks:

<TutorialStep step={5} title="Workflow Automation">

### Create a Simple Workflow

<CodeBlock language="bash" title="Create Workflow">
# Create a workflow for portfolio analysis
curl -X POST http://localhost:3000/api/workflows \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Portfolio Analysis Workflow",
    "description": "Automated analysis of portfolio documents",
    "steps": [
      {
        "type": "extract_text",
        "config": {
          "includeMetadata": true
        }
      },
      {
        "type": "ai_analysis",
        "config": {
          "prompt": "Analyze this portfolio and provide professional feedback",
          "model": "llama2:7b"
        }
      },
      {
        "type": "generate_summary",
        "config": {
          "maxLength": 200
        }
      }
    ],
    "triggers": {
      "fileTypes": ["md", "pdf", "txt"],
      "tags": ["portfolio"]
    }
  }'
</CodeBlock>

### Execute the Workflow

<CodeBlock language="bash" title="Execute Workflow">
# Execute workflow on a document
curl -X POST http://localhost:3000/api/workflows/{workflow-id}/execute \
  -H "Content-Type: application/json" \
  -d '{
    "documentId": "doc_1234567890_abc123"
  }'
</CodeBlock>

### Monitor Workflow Status

<CodeBlock language="bash" title="Check Workflow Status">
# Check workflow execution status
curl http://localhost:3000/api/workflows/{workflow-id}/status
</CodeBlock>

</TutorialStep>

## Common Issues and Solutions

<TroubleshootingSection>

### Issue: Upload Fails with "File Too Large"

**Solution**: Check your file size limits in the configuration:

<CodeBlock language="bash">
# Check current limits
curl http://localhost:3000/api/health | jq '.config.maxFileSize'

# Update in .env file
echo "STORAGE_MAX_FILE_SIZE=200MB" >> .env
</CodeBlock>

### Issue: AI Analysis Returns Empty Response

**Solution**: Verify Ollama is running and models are available:

<CodeBlock language="bash">
# Check Ollama status
curl http://localhost:11434/api/tags

# Pull required model if missing
ollama pull llama2:7b
</CodeBlock>

### Issue: Processing Stuck in "Processing" Status

**Solution**: Check the processing queue and restart if needed:

<CodeBlock language="bash">
# Check processing status
curl http://localhost:3000/api/health | jq '.services'

# Restart processing service
npm run restart:processing
</CodeBlock>

</TroubleshootingSection>
